Это файл с описанием данных, которые нужно получить с помощью 
ИИ-агента, и их формата

1. Словарь variables для ролей и переменных
Ключами будут либо названия ролей, либо названия переменных
Для роли в качестве значения будет словарь c ключом format со значением 'Executor' 
и с ключом editor со значением исполнителя, если он был указан, иначе - пустая строка
Для переменной в качестве значения будет словарь c ключом format со значением 
формата переменной (текст, дата и т. п.), для ключа editor будет в качестве 
значения пустая строка

variables = {
    lane1: {
        format: 'some format', 
        editor: 'some editor'
        },
    lane2: {
        format: 'some format', 
        editor: ''
        },
    variable: {
        format: 'some format', 
        editor: ''
        },
    ...
}


2. Словарь start_event для стартового события
2 ключа: name для названия события и value для связи с ролью

start_event = {
    name: 'startEvent_name',
    value: 'lane'
}

3. Словарь sequence_flows для переходов
В качестве ключей идут названия переходов (они должны быть уникальными), 
в качестве значений идет словарь с ключами 'from' и 'to' для значений id, которые 
должен соединять переход

sequence_flows = {
    flow1: {
        from: 'some_id',
        to: 'some_id'
    },
    flow2: {
        ...
    },
    ...
}


4. Список parallel_gateways для параллельных шлюзов
Названия шлюзов должны быть уникальными

parallel_gateways = [parallel_gateway1, parallel_gateway2, ...]


5. Словарь exclusive_gateways для исключающих шлюзов
В качестве ключей идут названия шлюзов, в качестве значений идет набор правил, 
связанных с исходящими переходами в виде строки (если правил нет, то идет пустая строка)

exclusive_gateways = {
    eg1: 'some_rules',
    eg2: '',
    ...
}

6. Словарь user_tasks для действий
В качестве ключей идут названия действий, в качестве значений идут словари, в которых
3 ключа со значениями-строками: value для связи с ролью (обязательный), 
deadline для дедлайна задания и reassign для булевого значения 
(необязательные, могут быть пустыми строками) 

user_tasks = {
    ut1: {
        value: 'some_lane',
        deadline: 'some_deadline',
        reassign: '',
    },
    ut2: {
        value: 'some_other_lane',
        deadline: '',
        reassign: true,
    },
    ut3: {
        value: 'one_more_another_lane',
        deadline: '',
        reassign: '',
    },
    ...
}


7. Словарь script_tasks для скриптов
В качестве ключей идут названия скриптов, в качестве значений идут словари 
с ключами value, принимающим значения "user.GetExecutorInfoHandler" или 
"var.FormulaActionHandler" и config со значением конфига в строковом виде

script_tasks = {
    st1: {
        value: "user.GetExecutorInfoHandler",
        config: 'some_config'
    },
    st2: {
        value: "var.FormulaActionHandler",
        config: 'some_another_config'
    }
}


8. Строковая переменная end_event для конечного события
Название конечного события 

end_event = 'some_end_event'

9. Словарь ids с значениями ID для всех элементов бизнес-процесса
Названия всех элементов бизнес-процесса уникальные (так как хранятся в словарях), 
поэтому они не перепутаются и каждому будет присвоен уникальный ID в виде целого числа

ids = {
    'some_lane': some_id,
    'some_sequence_flow': some_another_id,
    ...
}
